#  Splunk Sysmon PowerShell Investigation Lab

## Overview

In this lab, I performed a real-world SOC investigation using Splunk Enterprise and Sysmon logs collected from my Windows 11 endpoint through Splunk Universal Forwarder.

The objective was to understand how security analysts investigate PowerShell activity, correlate Sysmon events, analyze process execution, inspect command-line arguments, identify file creation events, review registry modifications, and determine whether activity is benign or malicious.

---

##  Objectives

- Configure Splunk Enterprise and Universal Forwarder
- Ingest Sysmon Operational Logs
- Investigate PowerShell process execution
- Analyze parent-child process relationships
- Review command-line activity
- Investigate file creation events
- Investigate registry modification events
- Practice event correlation methodology
- Develop SOC Tier-1 investigation skills

# Step 1 — Verify Sysmon Log Collection

Verified that Sysmon events were successfully ingested into Splunk.

### Query

```spl
index=* 
| stats count by source
```

### Result

```text
XmlWinEventLog:Microsoft-Windows-Sysmon/Operational
```

Confirmed that Sysmon Operational logs were being forwarded and indexed successfully.

# Step 2 — Investigate PowerShell Process Creation

### Query

```spl
EventID=1 Image="*powershell.exe"
```

### Purpose

Detect PowerShell execution events.

### Findings

Observed multiple PowerShell processes launched by:

```text
splunkd.exe
Splunk Universal Forwarder
SYSTEM
vivek
```

---

# Step 3 — Parent Process Analysis

### Query

```spl
EventID=1 Image="*powershell.exe"
| table _time Image ParentImage CommandLine
```

### Findings

Parent Processes:

```text
C:\Program Files\Splunk\bin\splunkd.exe

C:\Program Files\SplunkUniversalForwarder\bin\splunkd.exe
```

Process Tree:

```text
splunkd.exe
      ↓
splunk-powershell.exe
```

### Analysis

This activity was generated by Splunk services executing PowerShell scripts for internal monitoring and data collection.

Verdict:

```text
Benign Activity
```

---

# Step 4 — User Attribution Analysis

### Query

```spl
EventID=1 Image="*powershell.exe"
| stats count by User
```

### Findings

Users observed:

```text
SYSTEM

Splunkd

SplunkForwarder

vivek
```

### Analysis

- SYSTEM → Windows Services
- Splunkd → Splunk Enterprise Service
- SplunkForwarder → Universal Forwarder Service
- vivek → Interactive User Activity

This helps identify who executed a process.

---

# Step 5 — Command Line Investigation

### Query

```spl
EventID=1 Image="*powershell.exe"
| stats count by CommandLine
```

### Findings

Observed:

```text
powershell.exe

powershell.exe -ps2

powershell.exe -ExecutionPolicy Bypass

New-EventLog

wevtutil
```

### Analysis

PowerShell launched with:

```text
ExecutionPolicy Bypass
```

While commonly used by attackers, these commands were associated with legitimate Windows administration and Splunk operations.

Verdict:

```text
Suspicious Looking

But Legitimate
```

# Step 6 — File Creation Investigation

### Query

```spl
EventID=11
```

### Purpose

Investigate files created by PowerShell.

### Findings

Observed:

```text
__PSScriptPolicyTest_*.ps1
```

created by:

```text
powershell.exe
```

### Analysis

These files are automatically generated by PowerShell when validating script execution policies.

Verdict:

```text
Expected Windows Behavior
```

---

# Step 7 — Registry Modification Investigation

### Query

```spl
EventID=13
```

### Purpose

Detect registry modifications.

### Findings

Observed:

```text
svchost.exe
```

modifying:

```text
HKU\...\Store
```

### Analysis

Registry modifications were related to Microsoft Store configuration.

No persistence mechanisms detected.

Not observed:

```text
HKLM\Software\Microsoft\Windows\CurrentVersion\Run
```

Verdict:

```text
Normal System Activity
```

# Event Correlation

Instead of viewing events individually:

```text
EventID 1
EventID 11
EventID 13
```

I correlated them into a timeline.

### Investigation Flow

```text
PowerShell Process Created
        ↓
Command Line Executed
        ↓
File Created
        ↓
Registry Modified
        ↓
Final Assessment
```

This workflow mirrors real-world SOC investigations.
