#  Windows Process Creation Investigation Using Splunk

## Objective

The objective of this lab is to investigate Windows process execution events using Splunk and understand the SOC analyst investigation workflow.

This lab focuses on methodology instead of memorizing SPL queries.

---

## Investigation Methodology

```text
Alert / Suspicious Activity

          ↓

Identify Related Logs

          ↓

Analyze Timeline

          ↓

Find High Frequency Processes

          ↓

Investigate Interesting Processes

          ↓

Check User + Parent Process

          ↓

Analyze Command Line

          ↓

Decide Benign or Malicious
```

---

# Environment

**SIEM:** Splunk Enterprise

**Log Source:** Windows Security Logs

**Event ID Used:**

| Event ID | Description |
|---|---|
| 4688 | Process Creation Event |

---

# Investigation Scenario

A large number of process creation events were generated.

As a SOC analyst we investigate:

- What process executed?
- When did execution happen?
- Which account executed it?
- What parent process started it?
- Was CMD or PowerShell abused?

---

# Step 1: Verify Process Creation Logs

## Investigation Question

Are process execution logs available?

## SPL Query

```spl
source="WinEventLog:Security" EventCode=4688
```

## Finding

Splunk successfully collected Windows process creation logs.

Total Events Found:

```text
27,000+ Process Creation Events
```

## Analysis

Event ID 4688 confirms that process execution auditing is enabled.

Now we can investigate process activity.

---

# Step 2: Timeline Analysis

## Investigation Question

Was there a spike in process execution activity?

## SPL Query

```spl
source="WinEventLog:Security" EventCode=4688

| timechart span=5m count
```

## Finding

High process execution activity was identified.

Example:

```text
20:00 → 7991 Events
20:05 → 8304 Events
20:10 → 8395 Events
```

## SOC Thinking

A spike alone is not malicious.

Next question:

Which process generated this activity?

---

# Step 3: Identify Most Executed Processes

## Investigation Question

Which processes executed frequently?

## SPL Query

```spl
source="WinEventLog:Security" EventCode=4688

| stats count by New_Process_Name

| sort -count
```

## Finding

Top Processes:

```text
splunk-optimize.exe
postgres.exe
splunkd.exe
cmd.exe
GoogleUpdater.exe
conhost.exe
splunk-powershell.exe
```

## Analysis

Most activity was generated by Splunk services.

Processes selected for deeper investigation:

```text
cmd.exe
powershell.exe
```

Reason:

Attackers commonly abuse command interpreters.

---

# Step 4: PowerShell Investigation

## Investigation Question

Was PowerShell execution suspicious?

Important fields:

```text
Time
User Account
Parent Process
Command Line
```

## SPL Query

```spl
source="WinEventLog:Security"
EventCode=4688
New_Process_Name="*powershell.exe"

| table _time Account_Name Creator_Process_Name Process_Command_Line
```

## Finding

Observed Process Chain:

```text
splunkd.exe

      ↓

powershell.exe
```

Accounts:

```text
Splunkd
SplunkForwarder
```

## Analysis

PowerShell execution was started by Splunk related services.

No suspicious commands found.

Result:

```text
Benign Activity
```

---

# Step 5: CMD Investigation

## Investigation Question

Was cmd.exe abused?

## SPL Query

```spl
source="WinEventLog:Security"
EventCode=4688
New_Process_Name="*cmd.exe"

| table _time Account_Name Creator_Process_Name Process_Command_Line
```

## Finding

Observed:

```text
User:
vivek


Parent Processes:

svchost.exe

Splunk processes
```

Checked for suspicious parents:

```text
WINWORD.exe
chrome.exe
unknown.exe
```

Checked suspicious command behavior:

```text
Encoded Commands

Download Commands

Reverse Shell Commands
```

No malicious activity found.

# SOC Analyst Learning

This lab follows real investigation thinking:

```text
Something Executed

        ↓

What Executed?

        ↓

Who Executed It?

        ↓

What Started It?

        ↓

What Command Ran?

        ↓

Expected or Suspicious?
```

The goal of SOC investigation is not memorizing queries.

The goal is building investigation methodology.
