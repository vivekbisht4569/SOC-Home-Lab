# Wireshark Network Traffic Investigation

## Objective

The objective of this lab is to understand basic network traffic investigation using Wireshark from a SOC analyst perspective.

This lab focuses on analyzing network communication, identifying protocols, understanding packet flow, and building a structured investigation methodology.

The goal is not memorizing Wireshark filters.

The goal is understanding:

```text
Who communicated?

What protocol was used?

When did communication happen?

What evidence exists?

Is the traffic normal or suspicious?
```

---

```text
Network Alert / PCAP Capture
```

## ICMP Traffic

Command:

```cmd
ping google.com
```

Purpose:

Generate ICMP packets.

Network Flow:

```text
Windows Machine

        |

ICMP Echo Request

        ↓

Google Server


Google Server

        |

ICMP Echo Reply

        ↓

Windows Machine
```

SOC Purpose:

Verify connectivity and understand request/response communication.

# SOC Analyst Takeaway


Network investigation follows the same methodology as SIEM investigation.


```text
Packet Captured

        ↓

Who communicated?

        ↓

Which protocol?

        ↓

What happened?

        ↓

Find evidence

        ↓

Normal or Suspicious?
```


Wireshark is used to understand packets.

SIEM tools like Splunk use logs generated from network sensors.

The analyst skill is knowing what evidence to look for.
