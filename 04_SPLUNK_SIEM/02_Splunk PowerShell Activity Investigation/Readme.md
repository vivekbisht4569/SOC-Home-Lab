#  PowerShell Activity Investigation

##  Objective

The objective of this lab was to investigate PowerShell activity using Sysmon logs ingested into Splunk and learn how SOC analysts identify:

- Who executed PowerShell
- When it was executed
- What launched it
- Whether the activity was legitimate or suspicious

This lab simulates a common SOC investigation scenario where analysts are asked to review PowerShell activity on an endpoint.

---

#  Lab Environment

| Component | Purpose |
|------------|------------|
| Windows 11 | Endpoint |
| Sysmon | Endpoint Telemetry Collection |
| Splunk Universal Forwarder | Log Forwarding |
| Splunk Enterprise | Log Analysis |
| Event ID 1 | Process Creation Monitoring |

---

#  Background

PowerShell is one of the most frequently abused tools by attackers.

Threat actors commonly use PowerShell for:

- Downloading malware
- Executing scripts
- Persistence
- Lateral movement
- Privilege escalation

For this reason, PowerShell activity is often investigated first during SOC investigations.

---

#  Investigation Methodology

The investigation followed the standard SOC process:

```text
Identify Process
        ↓
Identify User
        ↓
Identify Parent Process
        ↓
Review Command Line
        ↓
Determine Legitimacy
```

---

# Step 1 – Identify PowerShell Executions

Query:

```spl
EventID=1 Image="*powershell.exe"
```

Purpose:

Retrieve all PowerShell process creation events.

---

# Step 2 – View Important Fields

Query:

```spl
EventID=1 Image="*powershell.exe"
| table _time User Image ParentImage CommandLine
```

Purpose:

Reduce noise and focus only on:

- Time
- User
- Process
- Parent Process
- Command Line

---

# Step 3 – Count PowerShell Executions

Query:

```spl
EventID=1 Image="*powershell.exe"
| stats count
```

Result:

```text
1381 PowerShell Process Creation Events
```

---

# Step 4 – Investigate Parent Processes

Query:

```spl
EventID=1 Image="*powershell.exe"
| stats count by ParentImage
```

Results:

```text
C:\Program Files\Splunk\bin\splunkd.exe
Count: 696

C:\Program Files\SplunkUniversalForwarder\bin\splunkd.exe
Count: 692

VS Code
Count: 1

CodeSetup.exe
Count: 5
```

Observation:

Most PowerShell executions were launched by Splunk services.

---

# Step 5 – Investigate Users

Field Analysis:

```text
NT SERVICE\Splunkd
Count: 686

NT SERVICE\SplunkForwarder
Count: 682

NT AUTHORITY\SYSTEM
Count: 7

LAPTOP-0VFRID7N\vivek
Count: 6
```

Observation:

Most PowerShell activity was generated automatically by Splunk services.

Only a small number of PowerShell executions were initiated by the logged-in user.

---

# Step 6 – Command Line Analysis

Observed Commands:

```powershell
powershell.exe
-NoLogo
-NoProfile
-NonInteractive
-ExecutionPolicy Bypass
```

---

# SOC Analyst Thinking

When analysts see:

```powershell
ExecutionPolicy Bypass
```

they immediately investigate because attackers commonly use this option.

Questions asked:

```text
Who launched PowerShell?
What process launched it?
Was the command encoded?
Was a file downloaded?
Was the execution expected?
```

---

# Investigation Findings

### User Activity

```text
User: LAPTOP-0VFRID7N\vivek
```

PowerShell activity was manually generated during lab testing.

---

### Parent Process

```text
VS Code
```

and

```text
splunkd.exe
```

were responsible for launching PowerShell.

---

### Command Line Review

No evidence of:

- Encoded Commands
- Download Cradles
- Invoke-WebRequest Abuse
- Malicious Scripts
- Obfuscated Commands

was identified.

---

# Final Assessment

### Verdict

```text
Benign Activity
```

Reason:

The majority of PowerShell executions originated from:

- Splunk Enterprise
- Splunk Universal Forwarder

The remaining executions were generated by the local user during lab activities.

No suspicious behavior was observed.

# Key Takeaway

This lab demonstrated how Sysmon Process Creation events can be analyzed within Splunk to investigate PowerShell activity.

By examining process relationships, users, and command-line arguments, a SOC analyst can determine whether PowerShell activity is legitimate administrative activity or potentially malicious behavior.

This workflow forms a fundamental skill used daily by SOC Analysts, Threat Hunters, and Incident Responders.
